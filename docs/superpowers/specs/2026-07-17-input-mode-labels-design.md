# 入力モードの初期表示・名称変更 仕様

## 目的

ハッカソンの主役である「長い会議・文字起こしを最大5テーマへ分割する体験」を、審査員が最初に迷わず試せるようにする。

## 表示順と初期状態

- 左側かつ初期選択を `会議・文字起こし` とする。
- 右側を `アイデアメモ` とする。
- 左から読む日本語・英語双方の利用者に、左側を主導線として提示する。

## 表示文言

### 会議・文字起こし

- 切替ボタン: `会議・文字起こし`
- 入力画面見出し: `会議・文字起こしからテーマを見つける`
- 説明: `議事録や会話ログを貼り付け、最大5つの判断テーマに分けます。確認後、テーマごとのDecision Traceを生成できます。`
- 最大入力: 30,000文字

### アイデアメモ

- 切替ボタン: `アイデアメモ`
- 入力画面見出し: `アイデアメモを整理する`
- 説明: `ひとつのアイデアや検討内容を、追跡できる6つの判断要素に整理します。`
- 最大入力: 12,000文字

## 動作

- 初回表示時は会議・文字起こし入力欄へフォーカスする。
- 切替後の既存機能、入力上限、API、生成結果、ZIP出力は変更しない。
- URLや保存済み状態によるモード復元は追加しない。

## テスト

- 初期状態で `会議・文字起こし` が選択され、30,000文字の案内が表示される。
- 切替ボタンは左から `会議・文字起こし`、`アイデアメモ` の順になる。
- `アイデアメモ` へ切り替えると12,000文字の案内が表示される。
- キーボード操作、単一メモ生成、複数テーマ生成の既存E2Eを維持する。
