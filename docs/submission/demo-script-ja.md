# 60〜90秒デモ台本

**0〜8秒｜入力画面**

「AIは答えを出せます。でも、なぜその答えになったのかは消えがちです。LiveNode Decision Traceは、答えではなく判断プロセスを残します。」

**8〜20秒｜サンプル選択・生成**

「今回は『町の自転車施策で実証地域を絞る』サンプルを選びます。メモには選択肢、312件の移動ログ、現場の制約、判断を変える条件まで入っています。」

［サンプルをクリックし、「Decision Traceを生成」を押す］

**20〜43秒｜6つの結果**

「結果は、状況、前提、判断基準、選択肢、推奨、次のアクションの6つ。結論だけでなく、利点・コスト・リスク・元に戻せるかまで追えます。」

［カードを短くスクロールする］

「ここが重要です。メモに書かれた事実には『入力からの根拠』、AIが補った部分には『AIによる推論』と表示されます。」

**43〜67秒｜KX Noteコピー**

「人が読むDecision Traceに加えて、AIがあとで再利用できるKX Noteにも変換できます。」

［「KX Noteをコピー」を押す］

「KX Noteは、主張、根拠、データ、制約、リンクの5部構成。明示されていないつながりは作りません。次のアクションも運用用に残ります。」

**67〜80秒｜締め**

「入力は保存せず、処理のためだけにOpenAIへ送信します。LiveNode Decision Trace。判断の結論ではなく、判断できる自分を残すためのデモです。」

## 応募説明・質疑用の位置づけ

中心メッセージは、次の一文です。

> 会話を、単なる決定事項ではなく「なぜそう判断したのか」を再利用できる記憶へ変える。

Decision Traceが残すのは、結論だけではありません。何を主張として考え、どの根拠を信じ、どんな制約を受け入れ、どの選択肢を退け、何が変われば判断も変わるのかを残します。これは人間を完全に再現する仕組みではなく、汎用LLMと本人の間に置く中間レイヤーです。未来の自分や別のAIが、一般的な高水準の平均だけでなく、その人の経験・価値観・判断軸を参照しやすくするための記憶です。

### 3つの層

- **Webデモ＝取り込み層（現在実装済み）：** 1つの日本語・英語の判断メモから、6項目のDecision Traceと2種類のコピー可能なMarkdownを生成します。
- **ローカルKX＝実働する精錬層：** 開発者が人間の確認を含むMarkdownの蒸留・精錬・再接続ワークフローとして実運用しています。Webデモに内蔵されているとは説明しません。
- **配布Skill＝将来の提供層：** 他の利用者やAI環境、Obsidian・Notion等の保存先へ届ける形は今後の予定です。自動保存や複数AIへの自動導入は未実装です。

### このフェーズで実装中の拡張

長い文字起こしから最大5テーマを検出し、人がテーマを確認・選択してからテーマ別Traceを生成し、Markdown ZIPとしてまとめる機能を実装中です。現時点のデモ撮影では、複数テーマ検出やZIPを実装済み機能として紹介しません。
